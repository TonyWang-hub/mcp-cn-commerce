# Internationalization (i18n)

mcp-cn-commerce supports Chinese (zh) and English (en) for all user-facing messages including error messages, CLI output, and log messages.

## Quick Start

### Setting Language via Environment Variable

Set `MCP_CN_COMMERCE_LANG` before starting any server:

```bash
# Use English
export MCP_CN_COMMERCE_LANG=en
mcp-cn-commerce health

# Use Chinese (default)
export MCP_CN_COMMERCE_LANG=zh
mcp-cn-commerce start oceanengine
```

### Setting Language Programmatically

```python
from i18n import t, set_language

# Switch to English
set_language("en")
print(t("error.missing_env_vars", vars="APP_KEY"))
# => "Missing required environment variables: APP_KEY"

# Switch to Chinese
set_language("zh")
print(t("error.missing_env_vars", vars="APP_KEY"))
# => "缺少必需的环境变量: APP_KEY"
```

## Translation Keys

Translation keys use dot notation organized by category:

### Error Messages (`error.*`)

| Key | Description |
|-----|-------------|
| `error.platform_name_empty` | Platform name validation |
| `error.platform_name_invalid` | Platform name format validation |
| `error.missing_env_vars` | Missing environment variables |
| `error.param_too_long` | Parameter length validation |
| `error.param_sql_injection` | SQL injection detection |
| `error.param_path_traversal` | Path traversal detection |
| `error.param_xss` | XSS pattern detection |
| `error.unknown_sign_method` | Unknown signing method |
| `error.batch_empty` | Empty batch request |
| `error.unknown_platform` | Unknown platform name |
| `error.api_error` | API error with code and message |
| `error.max_retries_exhausted` | Retry limit reached |

### CLI Messages (`cli.*`)

| Key | Description |
|-----|-------------|
| `cli.description` | Main CLI description |
| `cli.start_help` | Start command help |
| `cli.health_help` | Health command help |
| `cli.info_help` | Info command help |
| `cli.list_help` | List command help |
| `cli.version` | Version string |
| `cli.error_unknown_platform` | Unknown platform error |

### Health Status (`health.status.*`)

| Key | Description |
|-----|-------------|

| `health.status.importable_no_creds` | Importable but missing credentials |
| `health.status.not_ready` | Server not ready |

### Log Messages (`log.*`)

## API Reference

### `t(key, **kwargs) -> str`

Translate a message key to the current language.

- `key`: Translation key using dot notation (e.g., `"error.missing_env_vars"`)
- `**kwargs`: Format arguments interpolated into the template
- Returns the translated string, or the raw key if not found

### `set_language(lang: str) -> None`

Set the language for the current thread. Must be `"zh"` or `"en"`.

Raises `ValueError` for unsupported language codes.

### `get_language() -> str`

Get the current language. Checks thread-local setting first, then `MCP_CN_COMMERCE_LANG` environment variable, then defaults to `"zh"`.

### `reset_language() -> None`

Reset thread-local language setting, falling back to environment variable or default.

### `get_all_keys(lang=None) -> list[str]`

Get all available translation keys for a language.

### `get_translations(lang=None) -> dict[str, str]`

Get a copy of all translations for a language.

## Thread Safety

Language settings are stored in thread-local storage. Different threads can use different languages simultaneously without interfering with each other.

## Adding New Translations

1. Add the key to both `zh` and `en` dictionaries in `shared/i18n.py`
2. Use dot notation for the key: `category.subcategory.description`
3. Use `{placeholder}` syntax for format arguments
4. Run tests: `PYTHONPATH=shared pytest tests/test_i18n.py -v`
