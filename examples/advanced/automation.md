## 库存监控

### 低库存预警

```
请帮我设置库存预警：
- 当商品库存 < 10 时，发送提醒
- 每天早上 9 点检查一次
- 预警信息包含：商品名称、当前库存、最近 7 天销量
```

### 库存补货提醒

```
请帮我设置库存补货提醒：
1. 每天检查库存
2. 当库存低于安全库存时提醒
3. 提供补货建议（补货数量、补货时间）
```

### 滞销商品预警

```
请帮我设置滞销商品预警：
1. 每周检查一次
2. 标记 30 天无销量的商品
3. 提供处理建议（促销、清仓、下架）
```

## 订单监控

### 订单异常监控

```
请帮我监控订单异常：
- 超过 24 小时未发货的订单
- 退款率超过 5% 的商品
- 差评订单
```

### 订单时效监控

```
请帮我监控订单处理时效：
1. 待发货订单超过 24 小时提醒
2. 待处理退款超过 48 小时提醒
3. 物流异常订单提醒
```

### 大额订单监控

```
请帮我监控大额订单：
1. 订单金额超过 1000 元的订单实时提醒
2. 每日汇总大额订单
3. 大额订单的发货状态跟踪
```

### 广告花费监控

```

1. 每小时检查广告花费
```

```
1. 每天检查广告 ROI
2. 当 ROI 低于预期时提醒
3. 提供优化建议
```

### 广告预算监控

```
请帮我监控广告预算使用情况：
1. 各广告计划的预算使用率
2. 预算即将耗尽的广告计划
3. 预算使用异常的广告计划
```

## 异常告警

### 销售异常告警

```
请帮我设置销售异常告警：
1. 今日销售额较昨日下降超过 20% 时告警
2. 今日订单数较昨日下降超过 30% 时告警
3. 连续 3 天销售下降时告警
```

### 库存异常告警

```
请帮我设置库存异常告警：
1. 库存为 0 的商品告警
2. 库存突然下降超过 50% 告警
3. 库存数据异常（负数）告警
```

### 广告异常告警

```
请帮我设置广告异常告警：
1. 广告花费突然增加超过 50% 告警
2. 广告 ROI 突然下降超过 30% 告警
3. 广告计划突然停止告警
```

## 监控配置示例

### 库存监控配置

```json
{
  "monitoring": {
    "inventory": {
      "low_stock_threshold": 10,
      "check_frequency": "daily",
      "check_time": "09:00",
      "alert_channels": ["email", "wechat"],
      "alert_contacts": ["ops@example.com"]
    }
  }
}
```

### 订单监控配置

```json
{
  "monitoring": {
    "orders": {
      "unshipped_timeout_hours": 24,
      "refund_timeout_hours": 48,
      "large_order_threshold": 1000,
      "check_frequency": "hourly",
      "alert_channels": ["email", "sms"],
      "alert_contacts": ["ops@example.com"]
```

## 监控最佳实践

### 1. 设置合理的阈值

- 不要设置过于敏感的阈值，避免频繁告警
- 根据历史数据设置合理的阈值
- 定期调整阈值以适应业务变化

### 2. 选择合适的监控频率

- 库存监控：每天 1-2 次
- 订单监控：每小时
- 广告监控：每小时
- 销售监控：每天

### 3. 多渠道告警

- 邮件：适合详细报告
- 微信/短信：适合紧急告警
- 企业微信/钉钉：适合团队协作

### 4. 告警升级机制

- 初级告警：通知运营人员
- 中级告警：通知运营主管
- 高级告警：通知管理层

---

## 注意事项

1. **监控频率**：不要设置过于频繁的监控，避免对平台 API 造成压力
2. **告警疲劳**：避免过多的告警，确保每条告警都有价值
3. **数据延迟**：考虑平台数据延迟，设置合理的告警时间窗口
4. **权限控制**：确保监控系统有必要的 API 权限
